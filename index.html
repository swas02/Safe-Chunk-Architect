Lock File (.lock)

The "Single Access" guard. Prevents two app instances from fighting over the same data. It is the first thing checked on startup.

Manifest (manifest.json)

Stores Project ID, Human-readable name, and Software Version. Essential for ensuring data compatibility between updates.

Data Chunks

Fragmented project state. By saving in shards, we reduce the "Blast Radius" of any single file corruption to only ~10% of the project.

Backups (.bak)

The immediate safety net. Every successful save moves the previous state here. It is the recovery point for the 30-second intervals.

Checkpoints

Immutable historical versions. Created when a user manually clicks "Save Milestone." Unlike chunks, these are single, monolithic project snapshots.

Object Referencing (Memory Sync)

Python's Mutable Dictionaries ensure that UI updates are instantly reflected in the manager's memory. No sync code required.
